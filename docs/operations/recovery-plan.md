# 復旧計画

1. サービスを停止し、障害ファイルを読み取り専用で退避する。
2. Google Driveの「Wellnot DB Backup」から最新の検証済み日次バックアップを取得する。
3. Render Persistent Diskの `/var/data/wellnot/wellnot.sqlite` を別名へ退避し、取得したDBを同じパスへ配置する。
4. SQLite整合性検査、工事件数、工程件数、監査履歴の最終日時を照合する。
5. 管理者が代表案件の詳細・工程・原価・帳票を確認する。
6. 復旧時刻と失われる可能性のある期間を記録し、責任者承認後に再開する。

バックアップ間隔の既定値は24時間。保持期間、RPO/RTOは運用責任者が確定し、月1回は復元演習を行う。
