# システム設計書 SYS-0004

- 状態: proposed
- 設計責任: システム管理責任者
- 最終確認日: 2026-07-05

## 目的・要件・対象外

紙の工程票を前提に、工程の見える化、見積精度向上、原価管理、粗利改善を支援する。従業員は工程だけを更新し、金額・原価・実績集計・粗利・Excel出力を閲覧できない。管理者は全機能を利用する。

資金繰り、売掛、請求、入金は対象外。将来連携は不変な工事IDと明示的な連携境界で追加する。

## コンテキストと構成

```text
ブラウザ ─ HTTP/API ─ アプリケーション ─ 業務ルール
                         ├─ SQLite (DB-0003)
                         ├─ PDF用印刷HTML
                         └─ Excelテンプレートアダプター
```

画面、API、業務ルール、永続化、帳票を分離する。工程種別、現場注意事項、予算項目はマスタ／明細として固定列から切り離す。

## 主要業務ルール

- 案件登録時に工程マスタから工程を選び、順序と予定期間を持たせる。
- 図面番号管理は案件単位でON/OFF。ONの場合のみ工程を図面単位へ関連付けられる。
- 全工程完了で案件を完了状態へ移し、編集をロックする。
- 完了後は改善メモのみ管理者が入力できる。再編集は管理者のロック解除履歴を必須とする。
- 納期変更には理由を必須とし、変更前後・変更者・日時を保存する。
- 工数は工程実績メモから管理者が確定した時間のみ集計する。現場入力時には計算を求めない。
- 工程進捗は工程数の割合だけでなく、略称と状態記号（完了●、進行△、未着手○）を表示する。

## 品質設計

- 金額情報は表示で隠すだけでなく、従業員向けレスポンスから除外する。
- SQLはプレースホルダーを使用し、状態変更をトランザクション化する。
- 監査対象操作を追記専用履歴へ保存する。
- A4/A3帳票はCSSの印刷ページ定義を使用し、色だけに依存しない記号・罫線・パターンを併用する。
- 初期2か月のガント表示は日付範囲をURLで変更でき、横スクロールと月送りを提供する。

## 運用・復旧

SQLiteファイルは日次バックアップ対象とし、復元後に整合性検査と件数照合を行う。認証失敗、権限違反、ロック解除、削除、帳票出力を監査する。詳細は `docs/operations/recovery-plan.md`。
