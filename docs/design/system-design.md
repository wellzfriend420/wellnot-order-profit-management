# システム設計書 SYS-0004

- 状態: proposed
- 設計責任: システム管理責任者
- 最終確認日: 2026-07-12

## 目的・要件・対象外

紙の工程票を前提に、工程の見える化、見積精度向上、原価管理、粗利改善を支援する。単なる工程管理ではなく、社長・工場長・事務が利益を残すための適正経営を行う判断材料を見える化する。従業員は工程だけを更新し、金額・原価・実績集計・粗利・Excel出力を閲覧できない。管理者は全機能を利用する。

資金繰り・売掛回収管理は対象外。ただし、請負金額・請求予定・入金予定は受注工事明細と予実管理の判断材料として保持する。請求予定は案件に複数登録でき、取引先条件による自動計算と手動変更を併用する。

## コンテキストと構成

```text
PC/スマホ/タブレット ─ HTTPS共有URL ─ Render Web Service ─ 業務ルール
                                           ├─ Persistent Disk上のSQLite (DB-0003)
                                           ├─ Google Drive日次バックアップ
                         ├─ PDF用印刷HTML
                         └─ Excelテンプレートアダプター
```

画面、API、業務ルール、永続化、帳票を分離する。工程種別、現場注意事項、予算項目はマスタ／明細として固定列から切り離す。

## 主要業務ルール

- 案件登録時に工程マスタから工程を選び、順序と予定期間を持たせる。
- 図面番号管理は案件単位でON/OFF。ONの場合のみ工程を図面単位へ関連付けられる。
- 全工程完了で案件を完了状態へ移し、編集をロックする。
- 完了後は改善メモのみ管理者が入力できる。再編集は管理者のロック解除履歴を必須とする。
- 納期変更には理由を必須とし、変更前後・変更者・日時を保存する。
- 工数は工程実績メモから管理者が確定した時間のみ集計する。現場入力時には計算を求めない。
- 工程進捗は工程数の割合だけでなく、略称と状態記号（完了●、進行△、未着手○）を表示する。
- カレンダーはガントチャートではなく一般的な月間カレンダーとし、工事予定・人員予定を同じ予定データから画面内タブで切り替える。
- 予実管理は税込受注総額、見積原価、実績原価、見積粗利、実績粗利、利益差、粗利率を集計し、判断材料だけを表示する。
- 請求予定・入金予定は取引先マスタの締日・入金条件から自動計算し、実務例外と出来高請求のため手動変更・複数登録を許可する。

## 品質設計

- 金額情報は表示で隠すだけでなく、従業員向けレスポンスから除外する。
- SQLはプレースホルダーを使用し、状態変更をトランザクション化する。
- 監査対象操作を追記専用履歴へ保存する。
- A4/A3帳票はCSSの印刷ページ定義を使用し、色だけに依存しない記号・罫線・パターンを併用する。
- 約2か月の月間カレンダーは横スクロールを使わず、上下スクロールと月送りを提供する。

## 運用・復旧

本番SQLiteは `/var/data/wellnot/wellnot.sqlite` に固定し、永続ディスクへ保存する。1インスタンスで稼働し、SQLiteオンラインバックアップAPIで作った整合コピーをGoogle Driveへ日次保存する。認証失敗、権限違反、ロック解除、削除、帳票出力、手動バックアップを監査する。詳細は `docs/operations/production-deployment.md` と `docs/operations/recovery-plan.md`。
